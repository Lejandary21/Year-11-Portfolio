Overview

The design of the character that I would be thinking of is a 3D low poly design of a human, gender depends on the user as it will have a selection for them. The character would have the Jujutsu Tech school uniform. The character would have an average build and height.


End Users

The end users that I'm expecting are either Jujutsu Kaisen fans or Pokemon fans. The game I'm expecting is a fusion of the two. The age requirement for this game is 15 years old and above as the game might contain graphic content, anyone is free to try the game too. I will keep game in English first so English speakers can only play the game at the moment.

Requirements

  • Head
  • Specifications:
    • Average sized head
    • Prism shaped head
    • Lightskinned
  • Torso
  • Specifications:
    • Medium sized torso
    • More narrow shoulders
    • Slim waist
    • Rigged with a singple spine
    • Links to all other body parts
    • Lightskinned
  • Legs
  • Specifications:
    • Slender legs
    • Lightskinned
    • Rigged to walk/run
    • Links to the torso to be able to move
    • Lightskinned

Implications

  • Aesthetics

    Aesthetics is the visual representation of a character. Aesthetics is important for a character since It determines what the viewer thinks or feels about the character just based from the appearance of it.

  • I addressed this by:
    • The aesthetics that I've planned for the game is to have a dark environment. The animations would be 2d sort of similar to Generation 5 Pokemon animations to give it some sort of life and match with it's polygon-esk style of the game.

  • Fuctionality

    Functionality is the quality of something to be useful and practical for the purpose for which something was made. This is relevant to my character since it's going to be a low poly character, simple design simple mechanics.

  • I addressed this by:
    • The character should have a cartoony-esk animation style instead of fluid movements to match with the appearance of the character. By this, it will ensure it being functional and simple to use.
